# Epic: API Contracts & Security

Objective
- Define and implement core API contracts with strong security controls.

Acceptance Criteria
- OpenAPI spec for /chat, /status, /moderation finalized and versioned.
- Data models defined (ChatMessage, Session) with validation rules.
- OAuth2 and mTLS implemented; rate limiting in place.
- Tests cover endpoints, security, and error paths.
- Logging/trace hooks wired for observability.
